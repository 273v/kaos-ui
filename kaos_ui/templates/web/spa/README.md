# {{KAOS_PROJECT_NAME}}

A KAOS-native fullstack app: FastAPI on `kaos-core` (backend) +
Vite + React 19 + TanStack + Tailwind v4 + shadcn (frontend), wired
together with httpOnly-cookie auth, SSE chat streaming via
`kaos-agents`, and a Caddy reverse proxy that injects security
headers in production.

## Quickstart

```bash
make install           # uv sync (backend) + pnpm install (frontend)
cp .env.example .env   # then set APP_AUTH_TOKEN + your LLM API key
make doctor            # exits 0 on a fresh scaffold
make dev               # backend :8000 + vite :5173
```

The Vite dev server proxies `/v1/*` to the backend so cookies stay
first-party in development. Caddy handles the same job in production.

The frontend is pinned to `pnpm@11.1.0` and the workspace delays new
registry releases for 72 hours, blocks exotic transitive dependency
specifiers, and only allows reviewed dependency build scripts. After the
first `pnpm install`, commit `pnpm-lock.yaml`. In CI, use
`make install-ci && make verify-deps`.

## Routes

### Backend (`/v1/...`)

| Method | Path | What |
|---|---|---|
| `GET` | `/v1/health` | Liveness (no auth) |
| `GET` | `/v1/ready` | Readiness — deps reachable (no auth) |
| `POST` | `/v1/auth/login` | Body `{token}` → httpOnly cookie |
| `POST` | `/v1/auth/logout` | Clear cookie |
| `GET` | `/v1/auth/me` | Validate cookie |
| `POST` | `/v1/sessions/{id}/messages` | Body `{message}` → SSE stream |
| `GET` | `/v1/documents` | List |
| `GET` | `/v1/documents/{id}` | Read |
| `DELETE` | `/v1/documents/{id}` | Soft-delete |
| `GET` | `/v1/search?q=...` | BM25 search |
| `POST` | `/v1/uploads` | Multipart upload (size + magic-byte validated) |

OpenAPI: `http://127.0.0.1:8000/v1/docs` (dev only).

### Frontend (`/`)

| Path | What |
|---|---|
| `/login` | Token sign-in |
| `/_auth/chat` | Streaming agent chat |

The chat route is wired and demonstrates the SSE streaming pattern.
Add additional protected pages as `src/routes/_auth.<name>.tsx` —
see `CLAUDE.md` § How to add a frontend route.

## What ships safe by default

- **Refuse-to-start** in production with weak settings (token < 32
  chars, weak token literals, `APP_DEBUG=true`, wildcard CORS).
- **Magic-byte upload verification** via `python-magic`.
- **CWE-117** log-injection defense (CR/LF stripping in JSON formatter).
- **httpOnly + SameSite=Lax cookies** + `Origin` header allowlist on
  state-changing methods (belt-and-suspenders CSRF).
- **Caddy security headers**: CSP, X-Content-Type-Options, Referrer-
  Policy, Permissions-Policy, X-Frame-Options.
- **Node dependency hardening**: pinned pnpm 11.1, 72-hour dependency
  cooldown, exotic transitive dependency blocking, strict build-script
  allowlist, lockfile expected after first install.
- **Multi-stage non-root Dockerfile** with libmagic + urllib healthcheck.
- **Compose binds `127.0.0.1`** by default; production needs explicit
  override + Caddy in front.

See `CLAUDE.md` for the agent-editing rules and `kaos-ui/docs/SECURITY.md`
for the platform-wide contract.
