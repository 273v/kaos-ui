## What this project is

A fullstack web app scaffolded by `kaos-ui new web:spa`:

- **Backend**: FastAPI on `kaos-core` runtime + `kaos-agents` (chat
  via SSE), `kaos-content` / `kaos-pdf` / `kaos-office` (uploads),
  `kaos-llm-client` (provider transport).
- **Frontend**: Vite + React 19 + TanStack Router (file-based) +
  TanStack Query + Tailwind v4 + shadcn + Biome 2.x.
- **Caddy** terminates TLS, injects security headers, forwards SSE.
- **Auth**: bearer-token-from-`.env` ‚Üí httpOnly cookie. Same-origin
  via Vite dev proxy.

## Files that DO need updating when you add features

| Adding | Goes in | Don't forget |
|---|---|---|
| Backend route | `backend/app/routers/<name>.py` + `include_router` in `app/main.py` | a unit test using `TestClient` |
| Backend service | `backend/app/services/<name>.py` | lazy-import KAOS extras |
| Backend setting | `backend/app/settings.py` (`AppSettings`) | `.env.example` + `model_dump_redacted` if `SecretStr` |
| Frontend route | `apps/spa/src/routes/<name>.tsx` (or `_auth.<name>.tsx` for protected) | `tsc --noEmit` regenerates `routeTree.gen.ts` |
| Frontend component | `apps/spa/src/components/<Name>.tsx` (app-specific) or `packages/ui/src/components/...` (shared) | |
| New dep | backend: `cd backend && uv add ...`. frontend: `pnpm add ...` | |

Frontend dependency rules:

- Keep root `packageManager` pinned to `pnpm@11.1.0` or newer.
- Do not weaken `pnpm-workspace.yaml` supply-chain settings.
- Do not set `dangerouslyAllowAllBuilds: true`.
- Add dependency build scripts only to `allowBuilds` after reviewing why
  they need install-time execution.
- Commit `pnpm-lock.yaml` after the first install; CI should use
  `make install-ci && make verify-deps`.

## How to add a backend route

```python
# backend/app/routers/reports.py
from typing import Annotated
from fastapi import APIRouter, Depends
from pydantic import BaseModel
from app.deps import get_settings
from app.settings import AppSettings

router = APIRouter(tags=["reports"])

class Report(BaseModel):
    name: str
    rows: int

@router.get("", response_model=list[Report])
def list_reports(
    settings: Annotated[AppSettings, Depends(get_settings)],
) -> list[Report]:
    return [Report(name="example", rows=42)]
```

```python
# backend/app/main.py ‚ÄĒ register:
from app.routers import auth, documents, health, reports, search, sessions, uploads
app.include_router(reports.router, prefix="/v1/reports")
```

After a backend change, regenerate the typed frontend client: `make codegen`.

## Conventions inside this project

- **Auth flows**: only ever go through `apiFetch` / `apiJson` in
  `apps/spa/src/lib/api-fetch.ts`. They set `credentials: "include"`
  so the cookie attaches. Biome bans raw `fetch`.
- **Streaming**: use `readSseStream` from `apps/spa/src/lib/streaming.ts`.
  EventSource doesn't support cookies / custom headers the way we need.
- **Settings**: backend never reads `os.environ` directly ‚ÄĒ pull from
  `app.state.settings` via the `get_settings` Depends.
- **Logging (backend)**: use `kaos_core.logging.get_logger(...)` via
  the `app_logger("<sub>")` helper. JSON in prod, human in dev.
- **Errors**: backend raises `AppError` subclasses with
  `what / how_to_fix / alternative` content.
- **Heavy ops**: backend services lazy-import KAOS extras so the app
  boots without them.

## Auth swap path: bearer ‚Üí OIDC

The default bearer-from-`.env` is fine for a single-team internal
dashboard. For public-internet OIDC:

1. Replace `/v1/auth/login` body in `routers/auth.py` with an OIDC
   code-flow handler (`authlib`, `fastapi-users`, hand-rolled).
2. Issue a session cookie with the same shape ‚ÄĒ middleware unchanged.
3. Add CSRF middleware (double-submit token) for cross-origin tabs.
4. Update `tests/test_auth.py`.

Alternative: front Caddy with `oauth2-proxy`; trust `X-Forwarded-Email`.

## Troubleshooting

| Symptom | Fix |
|---|---|
| `SettingsError: APP_AUTH_TOKEN is not set` | `cp .env.example .env`, edit, restart |
| `SettingsError: ... is too short for production` | regenerate: `head -c 32 /dev/urandom \| base64` |
| `KeyError: "Attempt to overwrite 'X' in LogRecord"` | reserved key collision ‚ÄĒ rename. See kaos-ui PATTERNS.md |
| `TypeError: not all arguments converted` in logging | latent _HumanFormatter bug ‚ÄĒ see PATTERNS.md ¬ß Logging |
| Login OK but follow-up requests 401 | Check cookie `secure=` matches scheme. Dev/test = `false`; prod = `true` |
| 403 "Origin not allowed" | Add origin to `APP_CORS_ORIGINS`. Same-origin (proxy) deploys unaffected |
| Frontend can't see backend in dev | `VITE_BACKEND_URL=http://127.0.0.1:8000 pnpm dev` (override default port if backend runs elsewhere) |
| `pnpm codegen` fails | Backend must be running. Or point `openapi-ts.config.ts` at a static spec |
| SSE dropped mid-stream | Caddy `flush_interval -1` set? Some upstreams (Cloudflare) buffer too |
| `pnpm build` fails on `routeTree.gen` | Build script is `vite build` only ‚ÄĒ don't `tsc -b` first; use `pnpm typecheck` separately |
