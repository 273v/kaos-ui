# {{KAOS_PROJECT_NAME}}

A KAOS-native Streamlit dashboard with kaos-agents wired in by default.
Five pages, single-token auth, all backed by `kaos-core` runtime + VFS.

## Quickstart

```bash
make install          # uv sync + pre-commit hooks
cp .env.example .env  # then set APP_AUTH_TOKEN and your LLM API key
make doctor           # exits 0 on a fresh scaffold
make dev              # streamlit on http://127.0.0.1:8501
```

## Pages

| Page | What |
|---|---|
| Chat | Conversational agent loop via `kaos-agents.Runner` (chat pattern) with persistent SessionMemory |
| Upload | Parse PDFs / Office docs into a `ContentDocument` and stash it in VFS |
| Search | BM25 over every document in VFS |
| Browse | Per-document viewer + "open in chat" handoff |
| Settings | Resolved configuration (secrets redacted) + a doctor button |

## What ships safe by default

- `.env` is required; the app refuses to boot without `APP_AUTH_TOKEN`
- File uploads enforce 25 MB / extension allowlist server-side
- Streamlit error details are suppressed in `.streamlit/config.toml` —
  no stack traces leak to the browser
- Container is non-root, multi-stage, with a healthcheck
- Bind defaults to `127.0.0.1`; public exposure is opt-in

See `CLAUDE.md` for the agent-editing rules and the swap-path for
production auth (OAuth/OIDC).

## Production

```bash
# Edit .env to set APP_ENV=production and any secrets
docker compose up -d --build
```

For Postgres-backed state, use the overlay:

```bash
docker compose -f docker-compose.yml -f docker-compose.postgres.yml up -d
```

## Tests

```bash
make test
```

Covers: smoke (every page renders), auth flow, file upload validation,
and settings refusal in non-test environments without a token.
