# {{KAOS_PROJECT_NAME}}

A KAOS-native Textual TUI with `kaos-agents` wired in. Three screens
(Chat, Documents, Settings), all backed by `kaos-core` runtime + VFS.

## Quickstart

```bash
make install        # uv sync + pre-commit hooks
cp .env.example .env  # then set your LLM API key
make doctor         # exits 0 on a fresh scaffold
make dev            # full-screen TUI in this terminal
```

To debug widget layouts and runtime events live, open a second terminal
and run `make console` — Textual's dev console catches `app.log()` /
print output and shows the widget tree as you interact.

## Screens

| Screen | What | Default key |
|---|---|---|
| Chat | Conversational agent loop via `kaos-agents` Runner with persistent SessionMemory | `Ctrl+1` |
| Documents | Browse files in your VFS | `Ctrl+2` |
| Settings | Resolved configuration (secrets redacted) + key reference | `Ctrl+3` |

`Ctrl+Q` quits.

## What ships safe by default

- `.env` is required for any LLM provider key
- The app refuses to start in production with weak settings (debug
  on, missing keys, weak token literals)
- Logs go to `~/.kaos/{{KAOS_PROJECT_SLUG}}/log.jsonl` — Textual owns the
  terminal while running so logging never corrupts the UI
- Session memory persists across runs in `~/.kaos/{{KAOS_PROJECT_SLUG}}/vfs/`

See `CLAUDE.md` for the agent-editing rules and `kaos-ui/docs/SECURITY.md`
for the platform's full safety contract.
