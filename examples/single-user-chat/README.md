# single-user-chat

A working reference application demonstrating how to build a single-user agentic chat experience on top of `kaos-agents`, reusing the design system and SSE plumbing from the `kaos-ui` `web:spa` scaffold.

This is **not** a template ‚Äî it is documentation-by-code. A working app you clone, run, and read as the canonical "how the pieces fit together" example.

## 5-minute setup

Requires Python 3.13+, Node 24+, and `uv`/`pnpm` installed.

```bash
# 1. Generate a bearer token (32+ chars).
head -c 32 /dev/urandom | base64

# 2. Configure environment.
cp .env.example .env
# Edit .env: paste the token into KAOS_AGENTS_API_API_TOKEN
#   (note the DOUBLE 'API_' ‚Äî see docs/PATTERNS.md P-001)
# Set ANTHROPIC_API_KEY (or OPENAI_API_KEY, GOOGLE_API_KEY, XAI_API_KEY)

# 3. Install (stamps packages/ui + uv sync + pnpm install).
make install

# 4. Run.
make dev    # backend :8000, vite :5173 in parallel
# open http://localhost:5173/
# paste the bearer token to sign in
```

## What you get

- **Multi-session chat** with conversation persistence across reloads ‚Äî backed by `kaos-agents` `SessionMemory` + our metadata sidecar at `.kaos-vfs/single-user-chat/`.
- **Model picker** ‚Äî 10 current-generation models curated against `kaos_llm_client.cost.MODEL_PRICING` (Anthropic Haiku/Sonnet/Opus 4.x, OpenAI GPT-5/5.5, Google Gemini 2.5 Flash/Pro, xAI Grok 3).
- **Per-session system prompt + tool toggle** ‚Äî settings sheet on the right.
- **Transcript export** as Markdown or JSON ‚Äî pure client-side.
- **Streaming SSE** with all 15 kaos-agents wire events rendered: token deltas, intent classification, tool-call cards, usage chips, run-error inline messages, budget banners.
- **Editorial-legal design language** ‚Äî warm-stone OKLCH neutrals, near-black primary (NOT blue), amber accent, 1px hairlines, 8px radius, Inter + Source Serif 4. Cross-checked against Harvey, Legora, Midpage, Mike, and Claude.ai (see `docs/UX-LANGUAGE.md`).
- **`?debug=true` overlay** ‚Äî shows every raw event as it streams in.

## What it deliberately doesn't do (v1)

- No multi-user, no OIDC. Bearer-token-from-env only.
- No tool-call approval UI (kaos-agents emits the event; we banner-stub it).
- No RAG corpus management.
- No mobile-first responsive layout.
- Not published to PyPI.

See `docs/PRD.md` ¬ß Non-goals for the full list.

## Architecture in one paragraph

The backend mounts **`kaos_agents.api.server.create_app()`** ‚Äî that gives us session CRUD, SSE streaming, and bearer auth for free ‚Äî and layers a thin `/v1/chat/*` extension that owns our metadata sidecar (title, model, system_prompt, tools_enabled), proxies streamed messages with per-turn metadata applied as `MessageRequest` overrides, and exposes a curated model catalog at `/v1/models`. The proxy uses `httpx.ASGITransport` for same-process loopback ‚Äî no real network hop, works in `TestClient`. The frontend is a Vite SPA on TanStack Router + Query, with an exhaustive event-handler dispatch that produces deliberate state mutations for every one of the 15 kaos-agents wire event types and every observed `span(subject, phase)` combo. See `docs/ARCHITECTURE.md` for the full design.

## Troubleshooting

| Symptom | Fix |
|---|---|
| `InsecureApiConfigurationError: kaos-agents HTTP API refuses to start without an auth source` | Set `KAOS_AGENTS_API_API_TOKEN` (double `API_`!) in `.env`. See `docs/PATTERNS.md` P-001. |
| `KAOS_AGENTS_API_ALLOW_UNAUTH_LOCALHOST=1` has no effect | It's `KAOS_AGENTS_API_API_ALLOW_UNAUTH_LOCALHOST=1` ‚Äî same double-API quirk. |
| TestClient returns 401 with localhost-dev token | Localhost-dev mode rejects TestClient's remote address. Use `KAOS_AGENTS_API_API_TOKEN` instead in tests. P-002. |
| `pnpm typecheck` fails on `packages/ui/src/lib/api.ts` `import.meta.env` | Re-run `make sync-ui` ‚Äî it prunes unused sample stubs from the synced template. P-010. |
| `Conflicting configuration paths were found for the following routes: "/", "/"` | Make sure `_auth.sessions.tsx` exists before merging route changes. P-016. |
| Chrome DevTools MCP errors "Missing X server" | Add `--headless` to `~/.claude.json`'s `chrome-devtools` `args`. P-017. |
| Chrome DevTools MCP errors "The browser is already running" | `pkill -f chrome-devtools-mcp/chrome-profile && rm -f ~/.cache/chrome-devtools-mcp/chrome-profile/Singleton*`. P-018. |
